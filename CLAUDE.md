# Career Planner

## What this is
A local-first, CLI-based personal career planning tool.
Stores all data as flat Markdown/YAML files on the user's machine.
Optional AI coaching via BYO API key. MCP server for external integrations.

Think "Obsidian for your career."

## Tech stack
- Python 3.10+
- Typer (CLI framework)
- Rich (terminal UI, tables, trees)
- PyYAML (config/data parsing)
- httpx (HTTP client for LLM + HuggingFace API)
- Optional: FastMCP (MCP server), LanceDB (vector search)

## Key commands
- `pip install -e ".[dev]"` — install in dev mode
- `pytest` — run tests
- `career --help` — CLI entry point

## Code conventions
- Type hints on all function signatures
- All user-facing strings wrapped in `_()` for i18n (gettext)
- Flat file I/O goes through `src/career_planner/core/workspace.py`, never raw `open()`
- ESCO/O*NET data access through `src/career_planner/core/taxonomy.py`
- LLM calls go through `src/career_planner/core/llm.py` adapter, never direct provider calls
- Tests go in `tests/` mirroring the source structure

## Architecture
```
CLI commands (Typer)
  → src/career_planner/commands/*.py
    → src/career_planner/core/*.py (engine)
      → data layer (flat YAML/MD files via workspace.py)
      → taxonomy layer (bundled ESCO/O*NET via taxonomy.py)
      → LLM adapter (optional, via llm.py)
      → MCP server (optional, via mcp_server.py)
```

## Full specs
- `docs/blueprint.md` — architecture, data strategy, implementation plan
- `docs/man.md` — full man page with all commands and examples (user-facing, opened by `career man`)

## Workspace structure (created by `career init`)
```
my-career/
├── config.yml          # LLM provider, preferences
├── profile.yml         # Current role, target role, history
├── criteria.yml        # Job criteria (function, culture, growth, compensation, location)
├── skills/inventory.yml
├── brag/*.md           # Achievement entries (XYZ format)
├── resumes/*.pdf + *.yml
├── opportunities/*.md
├── assessments/*.md
├── conversations/*.md
└── data/coaching/{system-prompt.md, policies.md}
```

## Data files (bundled, do not edit by hand)
- `src/career_planner/data/esco-skills.yml` — curated ~1,000 tech-worker skills
- `src/career_planner/data/esco-occupations.yml` — ISCO groups 1-3 occupations
- `src/career_planner/data/esco-occupation-skills.yml` — occupation→skills mapping
- `src/career_planner/data/esco-skill-hierarchy.yml` — skill tree for browsing
- `src/career_planner/data/crosswalk.csv` — ESCO↔O*NET mapping
- `src/career_planner/data/transitions.yml` — JobHop transition probability matrix

These are generated by `scripts/prepare_*.py` (maintainer-only).

## i18n
- English (en) is source language, Vietnamese (vi) ships in v1
- Translation files: `locale/<lang>/LC_MESSAGES/career.po`
- All user-facing strings must use `_("string")` pattern
